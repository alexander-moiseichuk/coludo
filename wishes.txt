   - some sensor is primary put it current value as altitude: (ts0, val0)
   - if the same sensor pushed another value it will be come top again, old one kicked out if queue > MAX (e.g. 3 items)
   - if secondary sensor declare value it pushed primary value only if fresher for period P us: (ts1, val1), then (ts0, val0) otherwise (ts0, val0) stays actual
     and queue will be (ts0, val0) (ts1, val1) and here ts0 + P > ts1
   - if we have 3rd level sensor principle the same: if rank of sensor is the same - check for timestamp, otherwise comparison gap will be (ranks delta) * P

   we can ask sensor about declaring parameter for Blackboard and pointing expiration time, when adding parameter we add with expiration date.
   Every declaration will increase parameter max queue length by 1, so when adding will see ranks and expirations and simply compare. One extra value could be
   reseved for fusion extrapolation. That is probably more reasonable as each sensor knows that some values has one expiration period (or quality/trustiness), some another.

   We can even not use rank but deliver data and comare to expiration and Fusion will be take last value [0] if not expired and 2 last values [0]+[1]
   if value [0] already expired and extrapolate value to current time knowing that ts1 < ts0 linearly (no complex computations required)

   If you think any other options lets discuss but I wish to have first
